# Таблица User

Таблица, которая хранит пользователей. Поля:

- id - ид пользователя (PK)
- username - логин (уникальный)
- password - пароль (хешированный)
- email - почта (уникальная)
- created_at - дата создания
- last_login - дата последнего входа (может быть null, если пользователь ни разу не входил в систему)
- avatar_url - ссылка на картинку аватара
- balance - общий баланс пользователя (дробное число, но не может быть меньше 0)
- currency - валюта (строковое обозначение)

В каждом поле может быть только одно значение, поэтому выполняется НФ1.

`id → username, password, email, created_at, last_login, avatar_url, balance, currency`

Нет зависимости от части ключа, поэтому НФ2.

Все поля зависят от id напрямую, поэтому нет транзитивных зависимостей, поэтому НФ3.

В таблице только один первичный ключ (id), поэтому это НФБК.


# Таблица Budget

Таблица, которая хранит информацию о бюджете для каждого пользователя. Поля:

- id - ид бюджета (PK)
- title - заголовок бюджета
- description - описание бюджета
- created_at - дата создания бюджета
- start_at - дата начала бюджета
- end_at - дата конца бюджета (может быть null, если бюджет бессрочный)
- actual - стартовая сумма
- target - конечная сумма
- currency - валюта
- author - FK на таблицу User

В каждом поле может быть только одно значение, поэтому выполняется НФ1.

`id → title, description, created_at, start_at, end_at, actual, target, currency, author`

Нет зависимости от части ключа, поэтому НФ2.

Все поля зависят от id напрямую, поэтому нет транзитивных зависимостей, поэтому НФ3.

В таблице только один первичный ключ (id), поэтому это НФБК.

# Таблица WJT

Таблица, которая хранит информацию о refresh tokens. Поля:

- id - ид токена (PK)
- user_id - FK а таблицу User
- expired_at - дата истечения срока токена

`id -> user_id, expired_at`

Нет зависимости от части ключа, поэтому НФ2.

Все поля зависят от id напрямую, поэтому нет транзитивных зависимостей, поэтому НФ3.

В таблице только один первичный ключ (id), поэтому это НФБК.
